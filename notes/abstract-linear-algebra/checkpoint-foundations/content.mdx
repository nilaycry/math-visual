the first five notes built a complete picture of a single vector space. it's worth stopping
here to take stock of what that picture actually is before the next section changes the
question.

what you have: a vector space is a set where you can add and scale, governed by eight axioms.
inside it sit subspaces, the pieces that are themselves vector spaces. any subspace can be
described by a spanning list, and any spanning list can be compressed to a basis: the minimal
description that still reaches everything. every basis has the same size, and that size is
the dimension.

those aren't five separate facts. they're a single arc. dimension is meaningful because
bases exist. bases exist because spanning lists can always be compressed. spanning lists
can be compressed because of linear independence. linear independence is what you get when
a spanning list has no redundancy. each note was one step in the same argument.

---

at this point, the material often feels abstract in a particular way: you can follow the
definitions and proofs, but it isn't obvious what any of it is for. that's a reasonable
place to be. the purpose of these definitions isn't visible until you start using them to
study something.

the thing we're about to study is maps between spaces.

a linear map $T: V \to W$ is a function that carries the structure of $V$ into $W$.
the question shifts from "what is a space?" to "what can one space do to another?" and the
answer is constrained in precise ways by dimension. a map can't compress information without
accounting for it somewhere. it can't produce more than the space it starts from contains.
the vocabulary from the first five notes is exactly what makes those constraints precise.

if the first phase felt like collecting tools without a project, the next phase is the
project.
