# Math 314 Prose Style

Use this as the writing guardrail for Math 314 notebook notes.

## Goal

Write like someone who has already done the hard work of understanding the topic and is
now trying to make the structure visible for someone else.

The tone should be:

- rigorous
- compact
- personal enough to feel honest
- careful with definitions
- explicit about what is hard

## What good Math 314 prose usually does

- Starts from the mathematical distinction that actually matters.
- Says what kind of sentence or object is being discussed.
- Splits a theorem into hypotheses and conclusion when useful.
- Expands the exact definition that drives the proof.
- Names the non-obvious move instead of hiding it.

## Style rules

- Use lowercase titles.
- Use short paragraphs.
- Prefer one idea per paragraph.
- Define symbols before using them.
- Use first person where it helps honesty, not as filler.
- Do not use em dashes.
- Do not open with textbook filler like "In this note we will..."
- Do not overuse bold.

## Strong defaults

- Prefer "what is this statement really asking for" over generic motivation.
- Prefer "write the definition and see what it allows" over lofty summaries.
- Prefer "this is the step that matters" over polished but vague transitions.
- Prefer a small number of carefully chosen examples over a flood of examples.

## Things to avoid

- sounding like a textbook
- sounding like a motivational blog post
- pretending a subtle step is obvious
- piling three proof ideas into one paragraph
- ending sections by repeating what was just said

## Phrases and moves that fit this notebook

- "that distinction matters because..."
- "the first thing to notice is..."
- "the proof shape is already visible here."
- "pause here."
- "the hard part is not the algebra. it's knowing which definition to expand."
- "this is the point where the statement changes form."

## Phrases to avoid

- "it is important to note that"
- "in this section we will"
- "clearly"
- "obviously"
- "one can see that"
- "simply"

## Endings

Each note should end with a short honest boundary:

- what this note does not cover yet
- what next note it prepares for
- which subtle issue is being deferred

That keeps the sequence feeling cumulative instead of pretending every note is complete.
