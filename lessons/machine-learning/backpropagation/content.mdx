everyone says backpropagation is just the chain rule. they're right, but that answer never helped me the first few times I heard it. the chain rule on a formula is one thing. the chain rule running backward through a computation graph, reusing cached intermediate values, is something you have to see before it makes sense.

the [previous lesson](/lessons/first-network) set up the network and explained what the loss measures. this is where the gradients that training depends on actually come from.

---

### a tiny network

same network as the previous lesson: input `x`, hidden neuron (`w`, `b`), output neuron (`v`, `c`). now we extend the forward pass by one more step to include the loss:

<div className="math-highlight">z = w · x + b &nbsp;&nbsp;&nbsp; h = σ(z) &nbsp;&nbsp;&nbsp; y = v · h + c &nbsp;&nbsp;&nbsp; L = (y − target)²</div>

σ is tanh. the extra node `L` is the loss on a single training pair. computing how `L` changes when you nudge each of `w`, `v`, `b`, `c` by expanding the chain rule by hand gets exponentially messy. the graph approach keeps each local piece small.

the sketch draws this network as a computation graph. the forward pass runs left to right: watch the node values update as you drag `x` or the weights. the backward pass runs right to left, and that's where the gradients come from.

the colored bars on each node show the gradient magnitude at that node: how much a small change there would move the loss. bright means a lot of influence. dim means almost none.

the specific thing to look for: drag `x` to a large positive or large negative value, so the hidden neuron saturates. watch the gradient bar on `w` go nearly dark while the bar on `v` stays lit. the two weights are connected to the same loss, but one is getting almost no gradient signal. that's not a coincidence, and the next two sections explain why.

### the chain rule, but on a graph

the chain rule says: if `L` depends on `y`, and `y` depends on `h`, and `h` depends on `z`, then

<div className="math-highlight">∂L/∂z = ∂L/∂y · ∂y/∂h · ∂h/∂z</div>

on a graph, the version of this is: to get the gradient at any node, follow every path from that node to `L`, multiply the local derivatives along each path, and sum over paths. the local derivatives here are all simple:

<div className="math-highlight">∂L/∂y = 2(y − target)</div>
<div className="math-highlight">∂y/∂h = v</div>
<div className="math-highlight">∂h/∂z = σ'(z) = 1 − tanh(z)²</div>

multiply them to get `∂L/∂z`. then `∂L/∂w = ∂L/∂z · x`, and `∂L/∂b = ∂L/∂z · 1`.

pause here. this might feel straightforward on a network this small, but consider a network with a hundred layers. if you try to compute the derivative of the loss with respect to a weight in the first layer using classical calculus, you have to expand out the chain rule through every single path that connects that weight to the output. in a fully connected network, the number of intersecting paths grows exponentially. computing gradients by expanding the formula would require more operations than there are atoms in the universe.

what backpropagation actually is, is an elegant application of dynamic programming on a graph. it recognizes that millions of paths share the same sub-paths. instead of expanding the chain rule from the input forward, it computes the derivatives starting from the loss and flowing backward, caching the local gradient at every single node.

when it's time to compute the gradient for an early layer, it doesn't recalculate the chain. it simply multiplies its own local derivative by the cached downstream gradient that was literally just handed to it.

because each intermediate gradient is computed exactly once, the whole backward pass is O(n) in the number of parameters. it transforms a physically impossible combinatorial explosion into a fast, linear sweep. backpropagation isn't a new mathematical rule. it is an algorithmic mechanism that makes the chain rule computable on graphs with millions of nodes.

---

### why saturation kills learning

look at `σ'(z) = 1 − tanh(z)²`. when `z` is large, `tanh(z)` is near `±1` and `σ'(z)` is near `0`.

this is the crux. the backward pass multiplies by `σ'(z)` at each node it crosses. a saturated neuron nearly blocks the gradient entirely. that's what you saw in the sketch: drag `x` large enough and the `w` bar goes dark. the gradient isn't small. it's essentially zero. `w` will receive a near-zero update signal, step after step.

what makes this genuinely subtle is that the saturation can happen before training ever starts. if you initialize `w` to a large value, every input `x` immediately saturates the hidden neuron. the network is frozen at its starting position not because it's at a minimum, but because the gradient computation is multiplying by zero at the activation. gradient descent can't move a parameter whose gradient it can't see.

the saturation condition is `|tanh(z)| ≈ 1`, which means `|z| = |w·x + b|` is large. that happens whenever `w` is large relative to the typical input scale. this is why initialization matters: starting with weights drawn from a small distribution avoids saturating neurons before training begins.

you already saw this in the [previous lesson](/lessons/first-network). the "saturated init" preset showed the loss barely moving even after hundreds of steps. now you have the exact mechanism: `w`'s gradient was near zero from the start, because `σ'(z) ≈ 0` before training even began.

---

### why it matters

backpropagation is not a neural-network-specific algorithm. it's the chain rule applied to a computation graph, with caching to avoid redundant work. PyTorch, JAX, and TensorFlow are all doing this same backward pass automatically, on graphs with millions of nodes.

knowing the mechanics helps when training breaks down. when a loss stops improving and won't move, the gradient is probably vanishing before it reaches the weights that need to change. when it explodes and the loss goes to NaN, multiplications are accumulating rather than shrinking. knowing which layer or activation is the bottleneck is the difference between fixing it in an hour and blaming the architecture for a week.
