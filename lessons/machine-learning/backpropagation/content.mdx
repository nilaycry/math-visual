everyone says backpropagation is just the chain rule. they're right, but that answer never helped me the first few times I heard it. the chain rule on a formula is one thing. the chain rule running backward through a computation graph, reusing cached intermediate values, is something you have to see before it makes sense.

the [gradient descent](/lessons/gradient-descent) lesson assumes you already have the gradients. this is where they come from.

---

### a tiny network

take the simplest nontrivial network: one input x, one hidden neuron with weight w and bias b, one output neuron with weight v and bias c. the forward pass is a chain of four operations:

<div className="math-highlight">z = w · x + b &nbsp;&nbsp;&nbsp; h = σ(z) &nbsp;&nbsp;&nbsp; y = v · h + c &nbsp;&nbsp;&nbsp; L = (y − target)²</div>

σ is the activation function, the nonlinearity that lets the network represent more than a line. I'm using tanh here because its derivative is clean and bounded, which makes the gradients easier to follow visually.

L is a single number. the question is how L changes when you nudge w, v, or b. those are ∂L/∂w, ∂L/∂v, ∂L/∂b, and gradient descent needs all of them. computing them by expanding the full composition and differentiating by hand gets messy quickly. the graph approach keeps the pieces small.

### the chain rule, but on a graph

the chain rule says: if L depends on y, and y depends on h, and h depends on z, then

<div className="math-highlight">∂L/∂z = ∂L/∂y · ∂y/∂h · ∂h/∂z</div>

on a graph the version of this is: to get the gradient at any node, follow every path from that node to L, multiply the local derivatives along each path, and sum over paths. the local derivatives here are all simple:

<div className="math-highlight">∂L/∂y = 2(y − target)</div>
<div className="math-highlight">∂y/∂h = v</div>
<div className="math-highlight">∂h/∂z = σ'(z) = 1 − tanh(z)²</div>

multiply them to get ∂L/∂z. then ∂L/∂w = ∂L/∂z · x, and ∂L/∂b = ∂L/∂z · 1.

pause here. this might feel straightforward on a network this small, but consider a network with a hundred layers. if you try to compute the derivative of the loss with respect to a weight in the first layer using classical calculus, you have to expand out the chain rule through every single path that connects that weight to the output. in a fully connected network, the number of intersecting paths grows exponentially. computing gradients by expanding the formula would require more operations than there are atoms in the universe.

what backpropagation actually is, is an elegant application of **dynamic programming** on a graph. it recognizes that millions of paths share the same sub-paths. instead of expanding the chain rule from the input forward, it computes the derivatives starting from the loss and flowing backward, caching the local gradient at every single node ("memoization"). 

when it's time to compute the gradient for an early layer, it doesn't recalculate the chain. it simply multiplies its own local derivative by the cached downstream gradient that was literally just handed to it. 

because each intermediate gradient is computed exactly once, the whole backward pass is O(n) in the number of parameters. it transforms a physically impossible combinatorial explosion into a fast, linear sweep. backpropagation isn't a new mathematical rule; it is an algorithmic mechanism designed specifically to save calculus from computational collapse.

---

### why saturation kills learning

look at σ'(z) = 1 − tanh(z)². when z is large, tanh(z) is near ±1 and σ'(z)
is near 0. since the backward pass multiplies by σ'(z) at each node, a
saturated neuron nearly blocks the gradient entirely. that's what you saw in
the sketch: w barely moves because its gradient gets squeezed through that
near-zero derivative, while v stays lit because it sits after the saturation.

stack several saturated layers and this compounds. that story is in the
[gradient flow](/lessons/gradient-flow) lesson.

---

### scaling up is genuinely hard

in a deeper network the same mechanics apply at every layer, compounded. each additional layer multiplies the gradient by another local derivative. if those derivatives are consistently below 1, the signal reaching the early layers shrinks exponentially. if they're above 1, it explodes.

batch normalization, residual connections, and careful initialization all exist to keep gradients in a range where they stay usable. I think it's worth sitting with how non-obvious those were to find. they came from years of research on exactly this failure mode, not from theoretical necessity.

---

### why it matters

backpropagation is not a neural-network-specific algorithm. it's the chain rule applied to a computation graph, with caching to avoid redundant work. PyTorch, JAX, and TensorFlow are all doing this same backward pass automatically, on graphs with millions of nodes.

knowing the mechanics helps when training breaks down. when a loss stops improving and won't move, the gradient is probably vanishing before it reaches the weights that need to change. when it explodes and the loss goes to NaN, multiplications are accumulating rather than shrinking. knowing which layer or activation is the bottleneck is the difference between fixing it in an hour and blaming the architecture for a week.

---

### what happens next: training

one gradient on one example doesn't train a network. training means repeating the forward pass and the backward pass over a dataset, and at every step nudging each parameter a little in the opposite direction of its gradient. that nudging rule is [gradient descent](/lessons/gradient-descent), now applied to all four parameters at once instead of two.

the sketch below does exactly that. same network as before (x → z → h → y, with parameters w, b, v, c), but now there are six (x, target) points shown as teal dots, and the pink curve is the function the network currently represents. hit start training and watch the pink curve bend toward the teal one as the loss falls.

the four presets are the whole story in 30 seconds. try them in order. good init converges cleanly. saturated init starts with w=3, which pins tanh near ±1 on every data point, so ∂h/∂z ≈ 0 and w barely moves. that's the exact failure mode from the saturation section, now playing out live. lr too high makes the updates overshoot the minimum every step, so the loss oscillates instead of descending. lr too low takes steps so small that training visibly stalls even though everything is technically correct.
