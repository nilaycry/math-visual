the [previous lesson](/lessons/one-weight-to-many) showed that stacking layers is natural: the output of one weight matrix feeds into the next. you now have a model with real depth. the question is whether that depth actually buys you anything.

it doesn't, not yet. and the reason is worth understanding precisely before adding the fix.

---

### the failure of linear stacking

a matrix multiplication is a linear map: it stretches, rotates, or shears space. to build a more expressive model, the natural move is to chain them:

<div className="math-highlight">y = A(B(Cx))</div>

this looks like three layers of computation. it isn't. matrix multiplication is associative: `A(BC)x` is exactly `(ABC)x`. the chain collapses to a single matrix `M = ABC`. three layers of linear maps is still just one linear map.

this holds no matter how many matrices you stack. infinite layers of linear transformations represent nothing more than a single linear transformation. the model gains no new expressivity.

---

### breaking the collapse

the fix is to insert something non-linear between the matrix multiplications, so the chain can't collapse:

<div className="math-highlight">y = W₂ · tanh(W₁x + b₁) + b₂</div>

`tanh` is called an activation function. because it's non-linear, the equation can no longer be factored. `W₁` and `W₂` are forced to stay distinct. the multi-layer structure is preserved.

---

### the geometric role

a matrix transformation scales and rotates space. tanh bends it.

specifically, `tanh(z)` compresses extreme values toward `±1`, introducing curvature. chaining matrices and activations alternately rotates, stretches, and bends the input space repeatedly. structures that can't be separated by a flat hyperplane (like two interleaved spirals) can often be unfolded by enough of these steps until a flat final layer can slice them cleanly.

there's a theorem, the Universal Approximation Theorem, which formalizes this: a network with enough width and a non-linear activation can approximate any continuous function to arbitrary precision. I find the theorem more reassuring than actionable. it says the geometry you need exists somewhere in the parameter space, but says nothing about how to find it. that's what the next few lessons are about.
