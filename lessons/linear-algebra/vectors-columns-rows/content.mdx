most courses introduce vectors, then column vectors, then row vectors, and treat
them as basically the same thing written differently. they aren't. this note is
about what's actually going on.

---

### the abstract vector

a vector space is a set V together with rules for adding elements and scaling them
by numbers. elements of V are called vectors. that's it. no coordinates, no arrows,
no components necessarily.

the example worth holding in mind: the set of all polynomials of degree at most 2 is
a vector space. you can add polynomials, scale them, and the result is still a degree-2
polynomial. the elements are functions. they are still vectors in the mathematical sense.

an abstract vector exists before you pick any coordinate system. it's a point in a
structured space. there is no list of numbers attached until you make a choice.

---

### choosing a basis

a basis for `V` is a minimal spanning set: every vector in `V` can be written as exactly
one linear combination of the basis vectors. once you pick a basis `b₁, b₂, ..., bₙ`,
every vector `v` has a unique list of coefficients `(c₁, c₂, ..., cₙ)` such that
`v = c₁b₁ + c₂b₂ + ··· + cₙbₙ`.

that list of coefficients is the column vector. it is not the vector `v`. it is a
description of `v` relative to the chosen basis.

<div className="math-highlight">v &nbsp;↔&nbsp; [c₁, c₂, ..., cₙ]ᵀ &nbsp;&nbsp; (in the basis b₁, ..., bₙ)</div>

change the basis, and the column changes. the vector does not. the arrow in space stays
exactly where it is. its representation shifts.

this distinction is invisible in most first courses because everyone uses the standard
basis `e₁ = [1,0,...,0]`, `e₂ = [0,1,...,0]`, and so on. in the standard basis, the
coefficients coincide with the coordinates. so it's easy to believe the vector just is
its column representation. it isn't. the column is a description, and descriptions
are relative.

---

### the row

a column vector is a list of coefficients for a vector in V. a row vector is a
different kind of object entirely.

a row vector is a linear function from `V` to `R`. it takes a vector as input and returns
a single number. in coordinates, it multiplies each component by a weight and sums:
`f(x) = a₁x₁ + ··· + aₙxₙ`. the row `[a₁  a₂  ···  aₙ]` encodes those weights.

the row lives in the dual space `V*`. column vectors live in `V`. these are different
vector spaces that happen to have the same dimension and, in `Rⁿ`, look identical
(both are n-tuples of numbers). that's why the difference is so easy to miss.

I found this genuinely confusing for a while. the row vector and the column vector
look like the same object. you can mechanically convert one to the other by transposing.
but the transpose is not a rotation of the vector. it moves you between two different
spaces.

---

### what changes under a basis change

this is where the distinction becomes unavoidable.

suppose you rescale your basis: instead of the unit vector `e₁`, you use a basis vector
that is twice as long. the coordinate of a fixed point halves, because the unit of
measurement doubled. column vector coordinates scale inversely with the basis.

now take a linear function that returns value 3 for a unit x-input. in the new basis,
one unit of measurement corresponds to twice the old distance. so the function
returns 3 for an input at the old distance of 2. to describe this function in the
new coordinates, its coefficient needs to double: 6 times the new unit equals 3 times
the old unit times 2. row vector coefficients scale the same way as the basis.

column coordinates and row coordinates transform in opposite directions. this isn't
a quirk. it's the reason you can form the product `aᵀx` and get a basis-independent
number: the contravariant and covariant pieces cancel exactly. the dot product is
invariant under a change of basis precisely because one factor transforms one way
and the other transforms the opposite way.

for most purposes in a first course, this doesn't come up. once you start doing
geometry where the metric isn't the identity, or you change between coordinate systems,
you can no longer ignore which kind of object you're working with.
